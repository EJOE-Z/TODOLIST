# 智能 TODO 清单 · 使用说明

AI 驱动的任务、课表、提醒管理工具。支持本地多账号、课表导入、智能排程，可部署到公网供他人使用，也可安装到手机主屏幕或打包为 Android APK。

## 快速开始

### 环境要求

- Python 3.9+
- 现代浏览器（Chrome / Edge / Safari）
- 智谱 AI API Key（用于助手功能）

### 本地运行

1. 复制环境变量模板：

```bash
cd backend
copy .env.example .env
```

2. 编辑 `backend/.env`，填入：

```env
ZHIPU_API_KEY=你的智谱密钥
```

3. 安装依赖并启动：

```bash
pip install -r requirements.txt
python app.py
```

4. 浏览器打开：**http://localhost:5000**

> 若用 PyCharm「在浏览器中打开」会走 `63342` 等端口，现已自动连 `5000` 后端；仍建议直接用 **http://localhost:5000** 访问，数据与账号更稳定。

### 开发模式（前后端分离）

```bash
# 终端 1：后端
cd backend && python app.py

# 终端 2：前端
python -m http.server 8080
```

前端地址：http://localhost:8080

## 功能概览

| 模块 | 说明 |
|------|------|
| **助手** | 自然语言创建/修改/删除任务，导入课表，查看计划与周报 |
| **任务** | 待办列表、子任务、拖拽排序、番茄专注、任务日记 |
| **日历** | 按日查看任务/课程/提醒/日记，全部提醒与日记时间线 |
| **课表** | 周课表网格、单双周、学期第一周设置 |
| **日历** | 按日查看任务、课程与提醒 |
| **统计** | 完成率、趋势图表 |
| **设置** | 账号、通知、备份、安装 App |

顶部导航可切换模块；手机端底部有相同 Tab 栏。右上角可切换深色模式。

## AI 助手使用

在「助手」页输入框发送消息即可。支持多轮对话。

### 创建任务

示例：

- `帮我创建一个任务：周五前完成计算机网络实验`
- `添加软件工程作业，截止 6 月 10 日晚 19 点`
- `下周本课程上课前完成预习`

助手会引导你确认截止日期、子任务（可回复「无」或「跳过」），并自动关联课表中的课程。

### 管理任务

- `删除 xxx 任务`
- `完成 xxx`
- `把 xxx 截止时间改成 6 月 10 日`

### 课表相关

- `导入课表`（可上传 JSON / TXT / PDF）
- `这周怎么安排`
- `生成周报`

### 提醒

- `每晚 9 点提醒我复习`
- `查看所有提醒`

### 今日计划

助手会推送今日简报；卡片上的按钮可直接专注、推迟或完成任务。

## 任务管理

### 基本操作

- 点击 **+** 或助手创建任务
- 勾选复选框标记完成
- 拖拽调整顺序（待办与已完成区域可互拖）
- 点击任务查看详情、编辑子任务与截止时间

### 番茄专注

- 在任务详情或助手卡片中选择「开始专注」
- 默认 25 分钟倒计时

### 任务日记

- 在任务卡片点击 **📖** 图标为单个任务写日记
- **日历** 页点击 **「任务日记」** 查看日记（勾选「仅看选中日期」与左侧日历联动）
- 有日记的日期在月历上显示 **紫色圆点**
- 任务页 **「全部日记」** 按钮会跳转到日历

## 课表与日历

### 导入课表

1. 在助手说「导入课表」，或课表页点击导入
2. 支持 **JSON、TXT、PDF**
3. 导入后可在课表网格中查看，并自动参与任务关联与统计

### 课表设置

- **时间节次**：设置每节课的起止时间
- **学期第一周**：用于单双周计算

### 日历

- 点击日期查看当天任务、课程、提醒
- 颜色区分不同类型事件

## 提醒与通知

### 创建提醒

通过助手自然语言创建，或在「提醒管理」面板查看、编辑、删除。

### 开启系统通知

1. 打开 **设置 → 通知提醒**
2. 打开「系统通知」开关
3. 浏览器弹窗时选择 **允许**

关闭开关后不再推送，但不会撤销浏览器权限。若曾拒绝权限，需到浏览器站点设置中手动开启。

## 账号与数据

### 注册与登录

1. **设置 → 登录 / 切换账号**
2. 注册用户名 + 密码（至少 4 位）
3. 不同账号的数据**完全隔离**（任务、课表、提醒、记忆等）

### 修改资料

**设置 → 修改账号资料** 可更改用户名、密码。

### 数据备份

| 操作 | 路径 |
|------|------|
| 导出 | 设置 → 导出备份（JSON 文件） |
| 导入 | 设置 → 导入备份 |
| 清空当前账号 | 设置 → 清空当前账号数据（不可恢复，请先备份） |

> 数据默认存在**本浏览器 localStorage**。换设备或清缓存会丢失，请定期导出。

## 发布给他人使用

### 第一步：推送到 GitHub

```powershell
cd "项目目录"
git add .
git commit -m "更新"
git push
```

### 第二步：Render 云部署（推荐）

1. 注册 https://render.com ，连接 GitHub 仓库
2. **New → Web Service**，Runtime 选 **Docker**
3. 环境变量添加 `ZHIPU_API_KEY`
4. 部署完成后获得地址，例如：`https://todo-assistant-xxxx.onrender.com`
5. 将链接发给他人即可使用（不限 WiFi）

> 免费实例休眠后首次访问需等待约 30 秒唤醒。

### 他人如何安装到手机

| 平台 | 方式 |
|------|------|
| Android | Chrome 打开链接 → 设置 →「安装到手机 / 桌面」 |
| iPhone | Safari 打开 → 分享 →「添加到主屏幕」 |

### 数据说明

- 每人数据存在各自手机/浏览器，**互不影响**
- AI 功能共用服务器上的智谱 Key，请注意 API 用量

更详细的部署说明见 `backend/API.md`。

## 生成 Android APK

将应用打包为 `.apk` 安装包，他人可直接安装，无需应用商店。

### 前置条件

1. 已完成 **Render 公网部署**
2. 安装 [Android Studio](https://developer.android.com/studio)
3. 本机已安装 Node.js

### 步骤

1. 编辑 `js/native-config.js`，填入 Render 地址：

```javascript
window.__NATIVE_API_BASE__ = 'https://你的服务.onrender.com';
```

2. 安装依赖并同步：

```powershell
npm install
npm run cap:sync
npm run cap:open
```

3. 在 Android Studio 中：**Build → Build APK(s)**

4. 安装包路径：

```text
android\app\build\outputs\apk\debug\app-debug.apk
```

将 APK 发给他人，手机开启「允许安装未知来源」后即可安装。

## 常见问题

### 助手无响应 / AI 不可用

- 检查 `backend/.env` 或 Render 环境变量中 `ZHIPU_API_KEY` 是否正确
- 访问 `/api/health` 查看 `ai_configured` 是否为 `true`

### 修改任务后数据没变

- 确认已登录账号；刷新页面试试
- 若刚切换账号，需重新登录对应账号

### 手机无法安装 App

- 必须使用 **HTTPS** 公网地址（localhost 不行）
- iPhone 必须用 **Safari**，不能用微信内置浏览器

### APK 安装后 AI 不可用

- 检查 `js/native-config.js` 中的 Render 地址是否正确
- 手机需能访问该公网地址

### 清除浏览器数据后内容丢失

- 数据在本地，清缓存会删除；请定期 **导出备份**
